This supplemental manual will cover all you need to know in order to serve as an expert for your streaming bomb technician through the modern communication technology known as Twitch Chat.

Sending Messages to the Bomb

Each module will have an assigned numerical digit in order to identify between modules that may have similar appearance but different expected outcomes.
This will appear in the top-right corner of the message flyout as a command indicator of the form !x.
Start your message with that command and the remainder of the message will appear in the message flyout for that module.

Auto-play Mode

There is also the option to enable auto-play mode, which leaves the bomb entirely in the hands of you and your fellow Twitch Chat experts.
In order to interact directly with the bomb, your commands will need to be very specific, based on the module that you're interacting with.
See the details on the following pages on how to interact with each specific module.

Note: The Auto-play chat interpreter knows if a module is defused and will not interpret instructions for a defused module. Also, if given a sequence of instructions that causes a strike mid-way, it will stop the sequence mid-way to avoid further strikes.
However, it does not discern between different chat instructions, so coordinate with your fellow experts to avoid bad instructions to the bomb.

Module Command Reference (page 1)

Module Name Command & Example
Wires

Wire position is determined in order from top to bottom, first wire being 1.

To cut the wire at position 'x': <x>
e.g. !1 5

The Button

To press and immediately release: tap
e.g. !1 tap

To hold the button down: hold
e.g. !1 hold

To release the button with 'x' in the displayed time: release <x>
e.g. !1 release 4
Keypads

Buttons are expressed as their position on the module first from left to right, then top to bottom.

To press a sequence of buttons: <list>
e.g. !1 2 4 1 3

Simon Says

Colors can be expressed as either their English name (red, blue, green, yellow), or as their initial letter (r, g, b, y).

To press a sequence of buttons: <list>
e.g. !1 red red blue or !1 r r b
Who's on First

To press a button labelled as 'x': <x>
e.g. !1 ready
Memory

To press a button in the 'x' position: position <x>
e.g. !1 position 2

To press a button labelled as 'x': label <x>
e.g. !1 label 3
Morse Code

Frequencies are expressed as the three digits after the decimal point. e.g. 3.565 Mhz would be expressed as 565.

To transmit at 'x': <x>
e.g. !1 565

Module Command Reference (page 2)

Module Name Command & Example
Complicated Wires

Wire index is determined in order from left to right, first wire being 1; index does not count by the wire slots, but by the existence of wires.

To cut wires: <list>
e.g. !1 2 4 5 6

Wire Sequences

Wires are determined by the number slot that they are plugged in to.

To cut wires: <list>
e.g. !1 7 8

To move down a panel: down
e.g. !1 down

To move up a panel: up
e.g. !1 up
Mazes

Maze directions can be expressed as either their English name (up, down, left, right), or as their initial letter (u, d, l, r).

To navigate the maze with a sequence of movements: <list>
e.g. !1 up right down down left or !1 u r d d l
Passwords

Character column index is determined in order from left to right, first column being 1.

To cycle through the character column 'x': cycle <x>
e.g. !1 cycle 1

To try a password 'x': <x>
e.g. !1 world

Note: Submit will not be pressed unless the word can be spelled as specified.
